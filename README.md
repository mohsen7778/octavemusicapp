Octave is a high performance client side web music player built for modern users. It connects directly to public audio APIs and delivers a clean fast listening experience in your browser. No login. No backend. No tracking. Just music.

If you are tired of heavy streaming apps and want something fast private and actually yours this is it.

## What it does

Octave does not host or store any media. It acts as a client interface that fetches audio from public APIs in real time.

All playlists liked songs history and preferences are stored locally in your browser storage. Nothing is sent to any server controlled by this project.

## Features

Glassmorphism UI  
Dynamic ambient background based on album art colors.

Mobile First Design  
Optimized for phones with smooth animations and fast response.

Privacy by Design  
No backend no tracking no analytics. Everything stays on your device.

API Integration  
Uses public APIs like Invidious LRCLIB TheAudioDB and Wikipedia.

Sleep Timer  
Stops playback automatically after a selected duration.

Smart Shuffle  
Prioritizes songs based on your listening behavior.

Vault System  
Export and import your full data as JSON.

Auto DJ  
Generates continuous music recommendations based on your activity.

---

## Background Play

Octave works best with Brave Browser for background playback.

Most browsers stop audio when the tab is not active. Brave handles this better.

---

## How it works

Octave is a fully static web app.

User Input  
↓  
Octave Client  
↓  
Search → Invidious API  
Playback → YouTube IFrame API  
Lyrics → LRCLIB  
Artist Info → TheAudioDB and Wikipedia  
Storage → Browser localStorage  

No data is stored on any server owned by this project.

## Deploy Your Own

You should deploy your own instance for stable performance.

### GitHub Pages
1. Fork the repository  
2. Go to Settings → Pages  
3. Select branch and save  

### Cloudflare Pages
1. Go to pages.cloudflare.com  
2. Connect your repository  
3. Deploy with default settings  

### Netlify
Connect repo and deploy without build settings.
